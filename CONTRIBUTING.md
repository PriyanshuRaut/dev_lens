# 🤝 Contributing to DevLens

First off, thank you for considering contributing to DevLens! 

## 🍴 1. How to Fork
1. Click the **Fork** button at the top right of this repository.
2. Clone your forked repository to your local machine:
   ```bash
   git clone [https://github.com/YOUR_USERNAME/dev_lens.git](https://github.com/YOUR_USERNAME/dev_lens.git)
   ```
3. Navigate into the directory: `cd dev_lens`

## 🔀 2. How to Create a Pull Request (PR)
1. **Create a branch** for your changes: 
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your code changes and test them.
3. **Commit** your changes with a clear message: 
   ```bash
   git commit -m "feat: your concise commit message"
   ```
4. **Push** to your fork: 
   ```bash
   git push origin feature/your-feature-name
   ```
5. Go to the original DevLens repository and click **Compare & pull request**.

## 📜 3. Basic Rules
* Write clean, readable Flutter/Dart code.
* Ensure your commit messages clearly describe what you changed.
* Never commit directly to the `main` branch.
* If you are adding a new feature, update the `README.md` if necessary.
* Be respectful and constructive in discussions.
